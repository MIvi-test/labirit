### 1. **cells** (входные данные)
Общее количество ячеек в лабиринте = `columns × rows`  
Определяет размер решетки лабиринта.

### 2. **microseconds** (время на генерацию)

Время выполнения алгоритма в микросекундах. Измеряется между вызовом генератора и его завершением. Используется для оценки производительности алгоритма.

### 3. **all_pairs_connected** (связность каждой клетки с каждой)

Булево значение (true/false) — все ли клетки достижимы друг из друга через открытые проходы. Проверяется BFS-поиском из (0,0). Всегда true для корректного лабиринта.
Во всех клетках значение 1

### 4. **branch_ratio** (отношение разветвлений)

Доля ветвящихся ячеек от общего количества. Ячейка считается ветвью, если у неё 3+ соседей (степень >= 3).  
Формула: `branch_cells / total_cells`

### 5. **steps_count** (количество шагов)

Число операций, выполненных алгоритмом при генерации. Отслеживается метриками `labirit_metrics_snapshot()`. Показывает сложность алгоритма.

### 6. **memory_peak_bytes** (пиковое использование памяти)

Максимальное количество байт памяти, использованное алгоритмом генерации. Показывает требования к памяти для таких размеров. Статично относительно cells.

### 7. **wall_density** (плотность стен)

Доля закрытых стен от всех возможных стен. Формула: `closed_walls / (rows × columns × 4)`  
У каждой ячейки 4 стороны (top, right, bottom, left).

### 8. **dead_end_count** (количество тупиков)

Абсолютное число ячеек-тупиков (клеток с одним соседом). Ячейка-тупик имеет степень = 1.

### 9. **dead_end_ratio** (отношение тупиков)

Доля тупиков от общего количества ячеек. Формула: `dead_end_count / total_cells`

### 10. **corridor_ratio** (отношение коридоров)

Доля коридорных ячеек (клеток ровно с двумя соседями). Формула: `corridor_cells / total_cells`  
Коридоры — это простые проходы без разветвлений.

### 11. **avg_degree** (средняя степень)

Среднее число соседей на одну ячейку. Формула: `sum_of_degrees / total_cells`  
Диапазон 0-4 (каждая ячейка имеет макс. 4 соседа).

### 12. **cycle_count** (количество циклов)

Число циклических путей в лабиринте. Вычисляется по формуле теории графов:  
`cycles = open_edges - total_cells + connected_components`  
В идеальном дереве циклов = 0.

### 13. **diameter** (диаметр)

Максимальное расстояние между двумя ячейками в лабиринте. Вычисляется двойным BFS (алгоритм поиска диаметра графа).
**Алгоритм:**

- **Двухпроходный BFS** (алгоритм нахождения диаметра в дереве/графе):
  1. Стартуем BFS из произвольной вершины (в коде — вершина 0)
  2. Находим самую удалённую вершину `a`
  3. Запускаем BFS из `a`, находим расстояние до самой удалённой вершины
  4. Это расстояние и есть **диаметр** — максимальное кратчайшее расстояние между любыми двумя вершинами

### 14. **longest_path** (длинный путь)

То же самое, что и **diameter** — самый длинный кратчайший путь между двумя ячейками.

### 15. **symmetry_score** (оценка симметрии)

Оценка ротационной симметрии лабиринта (0-1). Сравнивает структуру стен ячейки с её отражением/поворотом на 90°, 180°, 270°.  
Высокое значение = сбалансированная, предсказуемая структура.
**Алгоритм:**

1. Для каждой ячейки берём 4 бита стен (top, right, bottom, left)
2. Сравниваем её с тремя "зеркальными" позициями:
   - Отражение по горизонтали (влево-вправо)
   - Отражение по вертикали (вверх-вниз)
   - Поворот на 180° (оба отражения)
3. Для каждой пары считаем **количество совпадающих битов стен** (от 0 до 4)
4. Суммируем все совпадения и делим на общее число сравнений

### 16. **fractal_dimension** (фрактальная размерность)

Фрактальная размерность стен лабиринта, вычисляемая методом box-counting (подсчет заполненных ящиков разного размера).  
Показывает сложность граничной структуры. Значения близки к 1-2.
**Алгоритм (Box-counting method):**

1. Представляем лабиринт в "укрупнённом" виде с размерами `(2*cols+1) × (2*rows+1)` (включая стены)
2. Для разных размеров блоков `box_size` (1, 2, 4, 8...):
   - Разбиваем сетку на блоки
   - Подсчитываем `boxes` — количество блоков, содержащих хотя бы одну стену
3. Строим регрессию: `log(boxes) = D × log(max_dim / box_size)`
   - `x = log(max_dim / box_size)`
   - `y = log(boxes)`
4. Фрактальная размерность `D` — это наклон линии регрессии

### 17. **aspl** (Average Shortest Path Length)

Средняя длина кратчайшего пути между всеми парами ячеек. Вычисляется O(V³) алгоритмом.  
Низкое значение = плотный, хорошо связанный лабиринт.
**Алгоритм:**

1. Для **каждой** ячейки (BFS от каждой вершины):
   - Вычисляем кратчайшие расстояния до всех остальных ячеек
   - Суммируем все расстояния
2. Общая сумма делится на `total × (total - 1)`
3. ASPL = (Σ\_{i≠j} dist(i,j)) / (cells \* (cells-1))

### 18. **entropy** (энтропия)

Энтропия распределения расстояний между всеми парами ячеек. Измеряется в битах.  
Высокая энтропия = разнообразное распределение расстояний.

1. Строим гистограмму расстояний между **всеми** парами ячеек (BFS от каждой вершины)
2. Для каждого расстояния `d` считаем количество пар `hist[d]`
3. Вычисляем энтропию Шеннона:
   H = - Σ p_d × log₂(p_d)

### 19. **benchmark** (бенчмарк)

Наименование алгоритма генерации: `prim`, `growing_tree`, `watson`, `dfs`, `binary_tree`.  
Метаинформация для идентификации результатов.

### 20. **aspect_ratio** (соотношение сторон)

Отношение размеров сетки: `columns / rows` или `rows / columns`  
Показывает, насколько лабиринт вытянут (квадратный = 1.0, прямоугольный > 1.0).
